# Рабочие данные (Generated Data)

## 📁 Описание

Эта папка содержит **сгенерированные рабочие файлы** с данными, которые были обработаны и дополнены.

### ⚠️ ВАЖНО

- **Все файлы в этой папке генерируются автоматически**
- **Не редактируйте файлы вручную** - все изменения будут потеряны при следующей генерации
- Оригинальные данные находятся в папке `../data/`

## 🔄 Как обновить данные

Запустите скрипт генерации:

```bash
node generate-working-data.js
```

## 📊 Структура файлов

### Справочники:
- `cities.json` - Города с реальным количеством заведений (`establishments_count`)
- `categories.json` - Категории заведений (копия из оригиналов)

### Файлы городов:
- `moscow.json`, `spb.json`, и т.д. - Данные по городам с добавленным полем `category_id`

## 🔧 Отличия от оригиналов

### 1. Города (`cities.json`)
Добавлено поле:
- `establishments_count` (integer) - реальное количество заведений в городе

**Пример:**
```json
{
  "id": 1,
  "name": "Москва",
  "slug": "moscow",
  "weight": 699875,
  "establishments_count": 294,
  ...
}
```

### 2. Файлы городов
Добавлено поле:
- `category_id` (integer|null) - ID категории заведения (1-12) или null если не определена

**Пример:**
```json
{
  "id": 4157,
  "name": "Московский компьютерный сервис Сеть",
  "category_id": 7,  // ← Услуги
  ...
}
```

## 🎯 Логика определения категорий

Категории определяются автоматически на основе:
1. **Название заведения** - ключевые слова (ресторан, бар, салон и т.д.)
2. **Features** - список особенностей заведения
3. **Описание** - дополнительные ключевые слова

Порог уверенности: минимум 3 балла для присвоения категории.

## 🔍 Ручная проверка

Заведения без категории (`category_id: null`) требуют ручной проверки. Это могут быть:
- Организации, не подходящие ни под одну категорию
- Заведения со слишком общими названиями
- Данные, требующие уточнения

## 📝 Для разработчиков

При изменении логики категоризации:
1. Обновите `CATEGORY_KEYWORDS` в `generate-working-data.js`
2. Перезапустите генерацию
3. Проверьте статистику распределения
