## 🔍 ВЫЯВЛЕННЫЕ ПРОБЛЕМЫ

### 1. ❌ Отсутствует поле `category_id`
**Проблема:** В файлах заведений нет поля `category_id`, что делает невозможным фильтрацию по категориям.

**Решение:** Разработан скрипт `generate-working-data.js`, который автоматически определяет и присваивает категории на основе:
- Ключевых слов в названии
- Ключевых слов в описании  
- Списка features заведения

### 2. ❌ Вес города ≠ Количество заведений
**Проблема:** Поле `weight` в городах не соответствует реальному количеству заведений.

**Примеры:**
- Москва: weight=699,875 vs фактически 294 заведения (разница 100%)
- СПб: weight=309,282 vs фактически 285 заведений (разница 99.9%)

**Решение:** При генерации добавлено новое поле `establishments_count`, которое содержит реальное количество заведений в каждом городе.

### 3. ⚠️ Категории не отображались
**Проблема:** На странице категорий были ошибки с выводом данных.

**Решение:** Обновлен `script.js` для корректной работы с рабочей папкой `data-generated/`.

## 🔄 КАК ПЕРЕГЕНЕРИРОВАТЬ ДАННЫЕ

```bash
node generate-working-data.js
```

Этот скрипт:
1. Проанализирует все 78 городов
2. Для каждого заведения определит категорию (1-12)
3. Добавит поле `category_id`
4. Посчитает реальное количество заведений
5. Сохранит результаты в `data-generated/`

**Время выполнения:** ~2-3 минуты
